PROJECT COMPONENT & ROLE ANALYSIS
===================================

1. OVERVIEW
-----------
This project is a hybrid research simulation combining "Agent-Based Logic Simulation" (Python) and "Network Layer Simulation" (NS-3 C++). Its purpose is to generate data and proofs for the research paper found in `paper.tex`.

2. PYTHON COMPONENTS (`src/` and Root)
--------------------------------------
The Python files serve three distinct roles: Logic Simulation, Visualization, and Utility.

A. LOGIC SIMULATION (`src/main.py`)
   - **Role**: Validates the "Decision Making" logic of the system.
   - **Theory**: Uses `simpy` (Discrete Event Simulation) to model nodes as agents.
   - **What it simulates**:
     - Sensor states (PIR trigger -> Thermal check -> Camera check).
     - Battery consumption (simple mathematical model).
     - "Voting" logic (asking neighbors for confirmation).
   - **Output**: Generates raw metrics and plots (e.g., `data/latency_cdf.png`).
   - **Key Dependency**: `src/config.py` (System constants), `src/models.py` (Node/Gateway classes).

B. REPORT GENERATION (`src/generate_report.py`)
   - **Role**: Compiles the final "Proof of Work" for the paper.
   - **Function**: 
     - Takes the plots generated by `main.py`.
     - Combines them with pre-written/hardcoded text (Executive Summary, Conclusion).
     - Exports a professional PDF: `output/simulation_report.pdf`.
   - **Note**: Some results tables in this script contain hardcoded "success" values, suggesting it acts as a template for the paper's final presentation.

C. UTILITIES (`extract_docx.py`)
   - **Role**: Data Extraction / Web Scraping helper.
   - **Function**: Extracts raw text from `.docx` files (e.g., `Layer - 02. Edge AI (1).docx`).
   - **Usage**: Likely used to pull content from draft Word documents to populate the LaTeX paper or analysis files.

3. NS-3 COMPONENTS (`ns3/`)
---------------------------
The NS-3 files serve the role of "Physical Network Verification".

A. NETWORK SIMULATION (`ns3/lora-dual-ring.cc`)
   - **Role**: Validates the "Communication" layer.
   - **Theory**: Simulates the physics of LoRa radio waves (868/915 MHz).
   - **What it simulates**:
     - Packet Collisions (what happens if two nodes talk at once).
     - Signal Attenuation (signal getting weaker over 23m).
     - Latency (exactly how long a message takes to travel air-to-air).
   - **Why it's needed**: The Python `simpy` simulation assumes messages just "arrive". NS-3 proves *if* they can arrive in a real wireless environment.

4. THE "SIMULATION TRAILS" WORKFLOW
-----------------------------------
The project implies a two-step validation process:

[Step 1: Python Logic Trial]
   Command: `python3 src/main.py`
   Goal: Prove that *if* the network works, the detection algorithm (voting) successfully filters false alarms.
   Result: Generates Graphs (PNGs).

[Step 2: PDF Report Generation]
   Command: `python3 src/generate_report.py`
   Goal: Package the graphs into a readable PDF format (`simulation_report.pdf`) to be attached to the paper.

[Step 3: NS-3 Network Trial] (Optional/Advanced)
   Command: `./ns3 run scratch/lora-dual-ring`
   Goal: Generate the low-level network statistics (latency/packet loss) cited in the paper.

5. SUMMARY OF ROLES
-------------------
| File/Dir | Technology | Key Responsibility |
|----------|------------|---------------------|
| `src/main.py` | Python (SimPy) | **Logic Verification**: "Does the algorithm work?" |
| `ns3/lora-dual-ring.cc` | C++ (NS-3) | **Network Verification**: "Can the signals reach?" |
| `src/generate_report.py`| Python (ReportLab) | **Documentation**: "Make it look professional." |
| `extract_docx.py` | Python (XML) | **Helper**: "Get text from Word docs." |
